遷移後、AIX 主機需要採取補救措施、以確保新匯入的 LUN 能夠被正確識別並正常運作。



== 遷移後 - 離線 FLI

對於離線 FLI，刪除 LUN 匯入並確認主機啟動器已連線後，將 LUN 上線。

NOTE: 請參閱作業系統廠商文件以取得最新和更新的命令清單。

.步驟
. 分離後重新掃描以確認乾淨狀態。
+
[source, cli]
----
cfgmgr
----
. 識別儲存裝置。
+
[source, cli]
----
lsdev -Cc disk
sanlun lun show all
sanlun lun show all -pv
lsmpio -l hdiskx
----
. 驗證裝置是否已連接。
+
[source, cli]
----
lspv |grep hdiskx
----
. 識別磁碟區群組和檔案系統。
+
[source, cli]
----
lspv
lsvg <vg_name>
lsvg -l <vg_name>
lsvg -p <vs_name>
----
. Varyonvg 和掛載檔案系統。
+
[source, cli]
----
varyonvg <vg_name>
mount /mount_point
df -g
----




== 遷移後 - 線上 FLI

刪除匯入關係後，確認啟用 UNMAP 後，新的 LUN 可以獨立運作。刪除 LUN 匯入後，檢查 LUN 是否已設定 `-space-reserve disabled`和 `-space-allocation enabled`。如果未設置，請進行相應修改。

[cols="4"]
|===
| 檔案系統 | 區塊裝置 | LVM （FS） | Sanboot 


| 步驟 1：重新掃描所有已對應 AIX 主機上的儲存設備 | 步驟 1：重新掃描所有已對應 AIX 主機上的儲存設備。 | 步驟 1：重新掃描所有已對應 AIX 主機上的儲存設備 | 步驟 1：重新掃描所有已對應 AIX 主機上的儲存設備 


|  |  | 如果裝置採用精簡配置，則會自動啟用 SCSI UNMAP。 | 如果裝置採用精簡配置，則會自動啟用 SCSI UNMAP。 
|===
.步驟
. 分離後重新掃描以確認乾淨狀態。
+
[source, cli]
----
cfgmgr
----
. 識別儲存裝置。
+
[source, cli]
----
lsdev -Cc disk
sanlun lun show all
sanlun lun show all -pv
lsmpio -l hdiskx
----
. 驗證裝置是否已連接。
+
[source, cli]
----
lspv |grep hdiskx
----
. 在裝置上驗證 UNMAP：
+
[source, cli]
----
chfs -a reclaim=normal /mount_point
lvmstat -v vg_name -r
----
