= ONTAP Foreign LUN Import（FLI）執行階段
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
執行階段是執行移轉的階段。它涵蓋執行離線或線上 Foreign LUN Import 以及將主機存取從外部陣列轉換到 ONTAP 陣列所需的端對端工作。



== ONTAP FLI 執行階段任務

以下是每個元件的 ONTAP FLI 執行任務。

[cols="1,2"]
|===
| 元件 | 工作 


| 主機  a| 
* 檢閱主機事件和系統記錄，以便及早發現問題並降低風險，避免進行任何重大重新配置。
* 在提前切換或修復之前，根據需要重新啟動主機以確保穩定性。
* 切換完成後，驗證主機存取。所有預期裝置均已存在，多重路徑功能正常，應用程式和檔案系統均已正常啟動。




| iSCSI FLI 和網路  a| 
* 確認 IC LIF 可到達外部陣列目標入口網站，並確保後端連線在整個匯入過程中保持穩定。
* 驗證 iSCSI 工作階段是否保持健全狀態，因為它們會驅動外部 LUN 探索和路徑可用性。




| 網路  a| 
* 確保已實施必要的分區和遮罩，以便 ONTAP 啟動器連接埠可以存取外部陣列 LUN 以進行探索和匯入。
* 切換完成後，根據需要更新區域，以便主機存取目標 ONTAP LUN，並為以後的清理做好準備。




| NetApp 儲存陣列（目的地）  a| 
* 在建立匯入工作之前、請確認 ONTAP 中已探索到外部 LUN。
* 確認探索後，建立匯入關係並啟動匯入工作。
* 監控匯入進度並處理例外狀況。視需要使用暫停和繼續來調節複本串流，尤其是在主機延遲增加時。
* 切換處理：
+
** 離線 FLI 轉換發生在匯入完成後。
** 線上 FLI 主機切換會提前發生，而匯入作業會在背景繼續進行。






| 外部儲存陣列（來源）  a| 
* 確保要遷移的來源 LUN 已正確呈現給 ONTAP 進行探索和匯入（根據傳輸方式，使用 FC 遮罩或 iSCSI 目標可見性）。
* 確保在匯入過程中主機無法存取來源 LUN。這一點至關重要，可以防止從多個來源執行寫入操作。
* 在線上切換期間，根據需要協調取消對應和重新對應動作，以防止雙重呈現，並確保主機順利轉換至 ONTAP。


|===
對於 FC 和 iSCSI 後端 FLI，執行成功與否很大程度上取決於後端連線的穩定性，即確保 FC/iSCSI 工作階段的健全狀況以及到外部 LUN 的完全路徑備援。由於主機 I/O（線上）和匯入複本可能會爭用相同的後端資源，因此必要時可以對匯入操作進行節流（例如，使用暫停和繼續）以降低主機延遲，並且移轉通常安排在非尖峰時段進行。
