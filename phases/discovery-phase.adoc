= ONTAP Foreign LUN Import (FLI) 探索階段
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP FLI 發現階段是移轉程序的第一步。收集有關來源環境、連線能力和目標 ONTAP 系統的所有必要資訊至關重要，以確保移轉順利進行。發現階段為移轉的規劃、準備、執行和驗證奠定了基礎。



== 使用 ONTAP FLI 為 iSCSI 遷移做好準備

ONTAP FLI 探索階段會收集完成移轉所需的技術詳細資料：

* 規劃移轉工作流程（線上與離線）
* 安全地執行主機補救
* 在建立匯入關係之前，請先驗證 ONTAP 上外部 LUN 的連線能力和探索情況


對於跨主機、NetApp 控制器和網路架構的結構化、可重複的資料收集，您可以使用 NetApp Active IQ OneCollect。OneCollect 旨在從儲存設備、主機、網路架構和交換器收集資訊，以進行移轉規劃和驗證。

對於 FC FLI，探索階段必須擷取外部陣列目標連接埠 WWPN。

對於 iSCSI FLI，探索階段必須：

* 擷取外部陣列目標入口 IP 和目標 IQN
* 驗證 IC LIF 連線能力
* 規劃全網狀 iSCSI 工作階段（每個 HA 節點連接到每個目標入口網站），以確保可以透過所有可用路徑發現外部 LUN




== ONTAP FLI 探索階段工作

每個元件都有一組工作來收集資料或驗證連線能力。

[cols="1,2"]
|===
| 元件 | 工作 


| 主機  a| 
* 確定主機作業系統版本、多重路徑堆疊和組態，以及任何特殊使用模式（例如檔案系統、原始裝置或叢集磁碟），以推動主機補救和轉換步驟。
* 擷取 LUN 清單，包括大小、識別碼或序號、分割區配置或位移，以及應用程式如何使用每個 LUN 以進行規劃和復原。
* 對於 FC 前端環境，請記錄主機啟動器 WWPN。如果適用於整體移轉和轉換設計，這些 WWPN 將用於分區和遮罩。
* 對於 iSCSI 前端環境，如果在移轉後使用，請記錄主機啟動器 IQN 和目標入口網站 IP，主機將在完全移轉後使用該 IP 進行登入。




| 網路架構與網路  a| 
* 對於 FC 環境：
+
** 記錄目前的分區（往返主機和來源陣列區域的分區），以便在完全移轉後可以安全地移除或更新。
** 擷取相關的交換器識別碼和連接埠。


* 對於 iSCSI 環境：
+
** 記錄 ONTAP IC LIF 與外部陣列目標入口網站之間的 VLAN、子網路和路由路徑
** 確認沒有防火牆或 ACL 阻止通訊。






| NetApp 儲存陣列（目的地）  a| 
* 確認 ONTAP 版本和 ECV 是否已準備好支援 iSCSI FLI。
+

NOTE: 如果叢集未達到支援層級，則會封鎖啟動器建立作業。

* 確定哪些節點將參與，例如擁有 LUN 的節點及其 HA 合作夥伴，並記錄這些節點上可用的叢集間 LIF。規劃備援；規劃兩個 HA 節點上的 IC LIF、啟動器和工作階段。
* 記錄 IC LIF IPspace 和子網路，並驗證它們是否可以存取外部目標入口網站。如果叢集中其他位置已在使用現有的 IC LIF，請避免在不同的子網路中建立新的 IC LIF。
* 驗證在規劃期間必須強制執行的目的地端限制、例如匯入大小上限、512B 區塊大小需求、以及與 iSCSI FLI 相關的工作階段和匯入擴充限制。




| 外部儲存陣列（來源）  a| 
* 識別外部陣列目標入口網站 IP 和目標 IQN；iSCSI FLI 不支援 iSCSI 探索工作階段，因此必須事先明確知道這些資訊。
* 確定對應到來源主機的特定 LUN，包括 LUN 大小和區塊大小。確保 LUN 穩定、未擴展，並且適合進行匯入規劃。
* 記錄主機存取和遮罩物件（例如主機和啟動器記錄），以便在轉換期間安全地取消對應或重新對應，並驗證復原準備就緒。
* 規劃並執行外部陣列資格認證。
+

NOTE: iSCSI FLI 需要現場認證；請勿假設通過 FC 認證的陣列就自動具備 iSCSI 的資格。



|===


== 建議的 ONTAP FLI 探索交付項目

* 單一的庫存表對應：主機 → 來源 LUN → 外部目標入口網站 / IQN → ONTAP IC LIF / 節點 → 目標 LUN，用於乾淨執行和回溯。
* 在變更執行之前擷取 OneCollect 套件（或同等套件），以便在發生問題時縮短分類時間。
