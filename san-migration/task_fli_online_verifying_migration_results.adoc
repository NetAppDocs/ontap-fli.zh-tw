= 驗證ONTAP FLI 線上遷移結果
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
將 LUN 從外部陣列遷移到ONTAP陣列後，外部 LUN 導入 (FLI) 功能可以對來源 LUN 和目標 LUN 進行逐區塊比較，以驗證遷移是否完整且準確。遷移驗證所需的時間與遷移過程大致相同（或略長）。

不需要進行遷移驗證，但強烈建議進行。

.關於此任務
* 從 ONTAP 9.17.1 開始，支援使用 FLI 離線遷移進行外國 LUN 的資料遷移，並支援  ASA r2 系統與其他ONTAP系統（ASA、 AFF和FAS）在儲存層實作方面有所不同。ASA r2 系統中，建立儲存單元（LUN 或命名空間）時會自動建立磁碟區。每個磁碟區僅包含一個儲存單元。因此，對於ASA r2 系統，您無需在 `-path`建立 LUN 時，請提供此選項；您應該包含儲存單元路徑。
* 遷移驗證會造成中斷。正在驗證的 LUN 需要在驗證期間處於離線狀態。


.步驟
. 離線待驗證的LUN。
+
[source, cli]
----
lun offline -vserver <SVM_name> -path <volume_path|storage_unit_path>
----
+
此範例顯示使 *flivol* 捲和 *fli_72C* SVM 中名為 *72Clun1* 的 LUN 離線的命令。

+
[listing]
----
cluster::*> lun offline -vserver fli_72C -path /vol/flivol/72Clun1
Warning: This command will take LUN "/vol/flivol/72Clun1" in Vserver "fli_72C" offline.
Do you want to continue? {y|n}: y
----
+
中斷時間從這裡開始。

. 開始LUN遷移驗證。
+
[source, cli]
----
lun import verify start -vserver <SVM_name> -path <volume_path|storage_unit_path>
----
. 監控驗證狀態。
+
[source, cli]
----
lun import show -vserver <SVM_name> -path <volume_path|storage_unit_path>
----
+
此範例顯示用於監控 *flivol* 捲和 *fli_72C* SVM 中名為 *72Clun1* 的 LUN 的驗證狀態的命令。

+
[listing]
----
ontaptme-fc-cluster::*> lun import show -vserver fli_72C -path /vol/flivol/72Clun1
vserver foreign-disk   path                operation admin operational percent
                                         in progress state state       complete
-------------------------------------------------------------------------------
fli_72C D0i1E+G8Wg6m   /vol/flivol/72Clun1 verify    started
                                                                        9
----
. 停止 LUN 驗證。
+
[source, cli]
----
lun import verify stop -vserver <SVM_name> -path <volume_path|storage_unit_path>
----
+
在將 LUN 重新連線之前，必須先明確停止 LUN 匯入驗證。否則，LUN 線上將會失敗。即使狀態顯示驗證已完成，也必須手動執行此步驟。

. 使 LUN 聯機。
+
[source, cli]
----
lun online -vserver <SVM_name> -path <volume_path|storage_unit_path>
----
+
中斷時間到此結束。
