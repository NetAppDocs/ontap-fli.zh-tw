= ONTAP FLI 離線遷移工作流程摘要
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
外部 LUN 匯入 (FLI) 資料遷移過程涉及多個關鍵步驟，以確保資料從第三方儲存陣列成功遷移到NetApp儲存系統。 FLI支援離線和線上遷移。在 FLI 離線遷移中，用戶端系統在從第三方外部儲存陣列向NetApp儲存系統遷移資料期間處於離線狀態。

.開始之前
* 您應該完成link:concept_migration_discover_phase_workflow.html["發現"] ， link:concept_migration_analyze_phase_workflow.html["分析"] ， 和link:concept_migration_plan_phase_workflow.html["規劃"]遷移過程的各個階段。
* 你應該link:configure-fc-adapter-initiator.html["為啟動器模式配置 FC 適配器"] 。
* 你應該link:concept_target_and_initiator_port_zoning.html["使用ONTAP儲存啟動器連接埠對外部陣列目標連接埠進行分割區"] 。


FLI 離線工作流程包括準備主機和外部 LUN 以供匯入、建立 LUN 匯入關係以及匯入資料。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["一"]link:prepare-host-offline-migration.html["準備你的主人"] 。
[role="quick-margin-para"]
在執行 FLI 離線遷移之前，您應該重新啟動主機並驗證主機多路徑配置。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["二"]link:prepare-foreign-lun-offline.html["準備外部 LUN"] 。
[role="quick-margin-para"]
在 FLI 離線遷移過程中，您必須從外部陣列採取手動步驟將外部來源 LUN 呈現給ONTAP儲存系統；然後，您必須採取額外的手動步驟來在ONTAP儲存系統上發現外部來源 LUN。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["三"]link:create-lun-import-relationship-offline.html["建立LUN導入關係"] 。
[role="quick-margin-para"]
為 FLI 離線遷移建立 LUN 匯入關係包括在ONTAP中將來源陣列 LUN 識別為外部 LUN、建立和設定目標磁碟區以包含外部 LUN、建立目標 LUN，最後建立匯入關係。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["四"]link:task_fli_offline_importing_the_data.html["從外部陣列匯入 LUN 數據"] 。
[role="quick-margin-para"]
使用 FLI 從外部陣列匯入 LUN 資料。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png["五"]link:task_fli_offline_verifying_migration_results.html["驗證遷移結果"] 。
[role="quick-margin-para"]
對來源 LUN 和目標 LUN 進行逐塊比較，以驗證遷移是否完整且準確。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-6.png["六"]link:remove-lun-import-relationship-offline.html["解除LUN導入關係"] 。
[role="quick-margin-para"]
FLI離線移轉完成後，可以安全地移除LUN匯入關係。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-7.png["七"]link:concept_fli_offline_post_migration_tasks.html["執行遷移後任務"] 。
[role="quick-margin-para"]
查看日誌中的錯誤，驗證主機多路徑配置，並執行應用程式測試以驗證遷移是否成功完成。
