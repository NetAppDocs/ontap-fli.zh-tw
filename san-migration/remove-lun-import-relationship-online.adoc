= ONTAP FLI 線上遷移後刪除 LUN 匯入關係
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
外部 LUN 匯入 (FLI) 線上遷移完成後，可以安全地移除 LUN 匯入關係。此時，主機正在存取新的NetApp陣列，執行對新ONTAP LUN 的所有 I/O，且來源 LUN 不再使用。

從 ONTAP 9.17.1 開始，支援使用 FLI 離線遷移進行外國 LUN 的資料遷移，並支援 link:https://docs.netapp.com/us-en/asa-r2/get-started/learn-about.html["ASA r2 系統"^].ASA r2 系統與其他ONTAP系統（ASA、 AFF和FAS）在儲存層實作方面有所不同。ASA r2 系統中，建立儲存單元（LUN 或命名空間）時會自動建立磁碟區。每個磁碟區僅包含一個儲存單元。因此，對於ASA r2 系統，您無需在 `-path`建立 LUN 時，請提供此選項；您應該包含儲存單元路徑。

.步驟
. 刪除LUN匯入關係。
+
[source, cli]
----
lun import delete -vserver <SVM_name> -path <volume_path|storage_unit_path>
----
. 驗證導入關係是否已刪除。
+
[source, cli]
----
lun import show -vserver <SVM_name>
----
. 或者，查看事件日誌以驗證遷移結果。
+
[source, cli]
----
event log show –event fli*
----
+
此範例顯示用於查看 FLI 遷移結果的事件日誌的命令的範例輸出。

----
